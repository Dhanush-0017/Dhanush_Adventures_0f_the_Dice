Sprint 2 — April 10–12, 2026 | Authentication

April 10, 2026 Sprint 2 Dhanush Nagarajan
Build

What was built today

  • Built login.php — POST form, session_start(), credential validation against users.txt
  • Built register.php — form validation, password_hash(), flat file write to data/users.txt
  • Built includes/functions.php — sanitizeInput(), loadUsers(), saveUser(), validateUsername(), validatePassword()

Design Decision

Used filter_input(INPUT_POST, ...) instead of $_POST[] directly throughout — this is explicitly required by the rubric for form security. Also used password_hash() with PASSWORD_DEFAULT for secure storage in the flat file.

Bugs / Issues

Initial issue with sticky form values not persisting on validation failure — fixed by storing $stickyUsername variable and echoing it back into the input's value attribute with htmlspecialchars().

Sprint 3 — April 13–15, 2026 | Game Board

April 13, 2026 Sprint 3 M. Abhinaya
Build

What was built today

  • Started game.php — PHP nested loop generating all 100 cells dynamically
  • Implemented zigzag board numbering (snake-style: row 10 = 91–100 right-to-left, row 9 = 81–90 left-to-right, etc.)
  • Built css/board.css — 10x10 CSS grid, cell styling, even/odd checkerboard pattern

Design Decision

The zigzag numbering formula uses $rowFromBottom % 2 === 0 to determine direction. Took time to get this right — standard Snakes & Ladders boards number from bottom-left to bottom-right then alternate. Verified against a physical board diagram.

Bugs / Issues

Initial board rendering had cell numbers in wrong order on odd rows. Fixed by reversing the column calculation: $cellNum = $rowFromBottom * 10 + (10 - $col) for odd rows.

Sprint 4 — April 13–16, 2026 | Dice Engine & Logic

April 15, 2026 Sprint 4 Dhanush Nagarajan
Build

What was built today

  • Built process_roll.php — POST handler for dice rolls, rand(1,6), processMove() function
  • Snake/ladder lookup using associative arrays from board_config.php
  • Bounce logic: if new position > 100, player stays in place
  • Session update: positions, dice history, move count, turn switch

Design Decision

All game logic is strictly server-side PHP — zero JavaScript. The dice roll is triggered by a POST form submission. This means every turn requires a page reload, which is intentional and matches the project requirement of no JS for game logic.

Bugs / Issues

Initial overshoot logic was setting position to 100 instead of keeping the player in place. Fixed based on proposal edge case: if roll would exceed 100, player stays at current position (bounce back).

April 16, 2026 Sprint 4 Dhanush Nagarajan
Build

What was built today

  • Built ai_turn.php and includes/ai_functions.php
  • AI Easy strategy — pure random rand(1,6)
  • AI Medium strategy — simulates all 6 rolls, picks best avoiding snakes (60% chance)
  • AI Hard strategy — two-turn lookahead with average expected position (75% chance)
  • AI Narrator engine — story-style messages using narrator_templates array
  • Move Probability Map — shows each possible landing cell with percentage

Design Decision

AI strategies use a probability factor (60%/75%) so they don't always pick the optimal roll — this makes the game feel more natural and less mechanical. Seeded rand() with turn number for reproducible narrator messages during debugging.

Lessons Learned

  • Building purely server-side without JavaScript is more restrictive than expected — every interaction requires a POST form and page reload. Planning the session state carefully at the start saves a lot of refactoring later.
  • The flat file approach for users.txt works well for small scale but would not scale to production — acknowledged in README as a course constraint.
  • CSS-only animations require careful use of ::after pseudo-elements and keyframes — no logic possible, purely visual. Kept it to transitions and keyframe animations that add polish without functionality.
  • CODD server file permissions for data/ folder need to be set correctly — this caught us during Sprint 2 and is worth noting for future deployments.
